Public string
  format
Details:
Alias:
Initial:
Stereotype:
Ordered:
Range:
Notes: human created format string with possible nesting e.g. zip:csv. <br /></p><p>Conventions on format strings:<br /></p><p><br /></p><p>file: mime-type or file extensions (for common tile types)<br /></p><p>Examples: csv (text/csv), xls (application/vnd.ms-excel or application/xls – there are about 6 alternatives!), html (text/html), pdf (application/pdf) etc<br /></p><p>api: {spec-type}+{mime-type-of-standard-response}<br /></p><p>Examples: sparql+rdf/xml, rest+json<br /></p><p>service: service/{service-identifier}/{type-of-object-or-file-format}<br /></p><p>Examples: service/gdocs/spreadsheet (google docs spreadsheet)<br /></p><p>It is very common for files to be provided in compressed form (e.g. zip, targz, tar). Strictly the format of this object is the format for the compression (e.g. zip). However, for users it is the underlying format that will matter. To solve this one provides the formats in nested order separate by a colon. Formats should be provide in outermost first (i.e. start with format of last layer and work inwards). Examples:<br /></p><p><br /></p><p>zip:json - a zipped json file e.g. myfile.json.zip<br /></p><p>targz:xml - an xml file that has been tar’d and gzipped e.g. myfile.xml.tgz<br /></p><p>torrent:zip:csv - a csv file that has been zipped and then provided as a torrent.<br /></p><p>It is common for a given API to provide data in multiple formats, for example xml and json. In this case use the ‘||’ term. Examples:<br /></p><p><br /></p><p>api/xml||json - an API providing both xml and json<br /></p><p><br /></p><p>It is common, at present, to find projects where the data is in lots of files with these files listed on an index page. Rather than attempt to create a resource entry for each file we have adopted the convention of creating a resource for the relevant index page with a special format string beginning “index”, e.g.:<br /></p><p><br /></p><p>index/html (an index page in html format)<br /></p><p>index/ftp (an index page for a ftp site)<br /></p><p>http://docs.ckan.org/en/ckan-1.8/domain-model-resource.html)<br /></p>

  url

Alias:
Initial:
Stereotype:
Ordered:
Range:
Notes: the key attribute of a resource (and the only required attribute). The url points to the location online where the content of that resource can be found. For a file this would be the location online of that file (or more generally a url which yields the bitstream representing the contents of that file – for example some “files” are only generated on demand from a database). For an API this would be the endpoint for the api. (http://docs.ckan.org/en/ckan-1.8/domain-model-resource.html)
